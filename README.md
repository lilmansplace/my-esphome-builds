# my-esphome-builds

Automated ESPHome firmware builds and OTA update manifests for my home device fleet.

[![Build ESPHome Firmware](https://github.com/lilmansplace/my-esphome-builds/actions/workflows/build.yml/badge.svg)](https://github.com/lilmansplace/my-esphome-builds/actions/workflows/build.yml)

---

## How It Works

```
Push to main / tag v*
        │
        ▼
GitHub Actions
  ├── Generate device matrix (all devices/*.yaml except device-* templates)
  ├── Spin up one runner per device (parallel)
  │     ├── Inject secrets.yaml from GitHub Actions Secrets  ← never committed
  │     ├── esphome compile devices/<name>.yaml
  │     └── Upload firmware/<name>.bin as artifact
  │
  └── On tag push (v*):
        ├── Download all firmware artifacts
        ├── Generate manifests/<name>.json for each device
        ├── Commit manifests back to main branch
        └── Create GitHub Release with all .bin files attached
```

Each device polls its manifest URL every 6 hours.
When the manifest `version` is newer than the running firmware, Home Assistant shows an **update available** badge — exactly like the HA Voice PE.

my-esphome-builds/
├── .github/workflows/
│   └── build.yml              # CI/CD workflow

│   ├── webserver.yaml         # Web server auth (uses !secret)
│   ├── ntp.yaml               # SNTP time sync

├── components/
│   └── si4703/                # Custom FM radio chip driver

> Public adoption templates live in the private HomeAssistant repo and are community resources.

## Secrets Setup

**No credentials are ever stored in this repo.**

WiFi credentials, OTA passwords, and API keys are stored as [GitHub Actions Secrets](https://docs.github.com/en/actions/security-guides/encrypted-secrets) and injected into `secrets.yaml` at build time only.

Add these 5 secrets in **Settings → Secrets and variables → Actions**:

| Secret Name | Description |
|---|---|
| `ESPHOME_WIFI_SSID` | Home WiFi network name |
| `ESPHOME_WIFI_PASSWORD` | Home WiFi password |
| `ESPHOME_WEB_SERVER_PASSWORD` | ESPHome web server auth password |

> No OTA password or API encryption key — devices run open on a trusted local network.
> The `api:` and `ota:` components in `includes/common.yaml` have no password set.

## Releasing a New Build

```bash
git tag v2025.4.0
git push origin v2025.4.0
```

GitHub Actions will:
1. Build firmware for all ~58 devices in parallel
2. Generate OTA manifests pointing to the release assets
3. Create a GitHub Release with all `.bin` files attached
4. Commit updated manifests back to `main`

Devices will detect the update within 6 hours (or sooner after a restart).

## First-Time Setup: Adding OTA Updates to Devices

Run once from repo root to add the `update:` package to all device YAMLs:

```bash
# Preview changes without writing
python3 scripts/add-update-component.py --dry-run

# Apply to all eligible device YAMLs
python3 scripts/add-update-component.py
```

After running, build and manually OTA-flash each device once to deploy the
firmware that includes the update component. All subsequent updates will be
handled automatically via HA.
